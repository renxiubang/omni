## 用户需求概述

为现有 Omni Web Chat 应用增加多用户登录模拟系统、单词解释模块和用户单词本模块。

## 产品概述

基于现有微信风格 Web 智能体对话应用，新增用户系统和学习辅助功能。用户可以通过简单用户名输入模拟登录，在对话过程中选中英文文本获取单词详细解释（含音标、语音播放、例句），并将生词加入个人单词本进行统一管理。

## 核心功能

### 1. 模拟多用户登录

- 应用启动时显示登录页面，用户输入用户名即可登录（无密码验证）
- 登录状态通过 localStorage 持久化存储
- 支持退出登录功能（在设置界面添加退出登录按钮）
- 用户名唯一标识用户

### 2. 单词解释模块

- 用户在聊天界面选中英文文本时，自动弹出浮框显示详细解释
- 解释内容包含：
- 中文解释（词性 + 释义）
- 读音音标（IPA格式，英式和美式）
- 语音播放按钮（调用TTS API播放单词发音）
- 例句（英文例句 + 中文翻译）
- 浮框右上角显示五角星按钮，点击可将单词加入个人单词本
- 浮框支持关闭，不阻碍用户继续操作

### 3. 用户单词本模块

- 基本的增删查功能
- 单词列表按添加时间倒序排列
- 支持删除已收藏的单词
- 显示单词、音标、释义、例句、添加时间

### 4. UI 入口

- 聊天页面右上角添加单词本图标按钮
- 点击按钮进入单词本列表页面
- 设置界面添加退出登录按钮

## 技术栈选择

### 前端技术栈

- **框架**: React 19 + TypeScript（与现有项目一致）
- **样式**: Tailwind CSS v4（与现有项目一致）

- **状态管理**: React Context（用于用户认证状态）

#### API端点设计

**用户相关API：**

- `POST /api/users/login` - 模拟登录（创建或获取用户）
- 请求体：`{"username": "string"}`
- 响应：`{"id": 1, "username": "xxx", "created_at": "..."}`

- `GET /api/users/me?username=xxx` - 获取当前用户信息
- 响应：`{"id": 1, "username": "xxx", "created_at": "..."}`

**单词本相关API：**

- `POST /api/wordbook/add` - 添加单词到单词本
- 请求体：`{"user_id": 1, "word": "hello", "phonetic_uk": "...", "phonetic_us": "...", "explanation": "...", "example_sentence_en": "...", "example_sentence_zh": "..."}`
- 响应：`{"id": 1, "word": "...", ...}`

- `GET /api/wordbook/list?user_id=xxx` - 获取用户单词本列表
- 响应：`[{"id": 1, "word": "...", ...}, ...]`

- `DELETE /api/wordbook/{word_id}` - 删除单词本中的单词
- 响应：`{"success": true}`

**单词解释API：**

- `POST /api/wordbook/explain` - 获取单词/词组的详细解释
- 请求体：`{"word": "hello"}`
- 响应：`{"word": "hello", "phonetic_uk": "...", "phonetic_us": "...", "explanation": "...", "example_sentence_en": "...", "example_sentence_zh": "..."}`
- 实现：调用现有 `omni_client` 使用大模型API生成解释

**单词发音API：**

- `POST /api/wordbook/pronounce` - 获取单词发音音频
- 请求体：`{"word": "hello", "voice": "female"}` (voice可选)
- 响应：音频文件（WAV格式）
- 实现：调用DashScope TTS API生成单词发音

### 4. 单词解释实现逻辑

#### 文本选中监听

- 在 `ChatPage` 组件中监听 `mouseup` 事件
- 使用 `window.getSelection()` 检测是否有文本被选中
- 判断选中文本是否为英文（通过正则 `/^[a-zA-Z\s\-']+$/` 匹配）
- 获取选中文本的位置信息（`getBoundingClientRect()`）用于定位浮框
- 防抖处理：使用 `setTimeout` 延迟300ms显示浮框，避免频繁触发

#### 解释生成

- 调用后端 `POST /api/wordbook/explain` 接口
- 后端使用现有 `omni_client` 调用大模型API生成解释
- Prompt 设计：

```
请详细解释以下英文单词/词组，包括：
1. 词性和中文释义
2. IPA音标（英式和美式）
3. 英文例句和中文翻译

请按以下JSON格式返回，不要返回其他内容：
{
  "explanation": "词性 + 中文释义",
  "phonetic_uk": "英式IPA音标",
  "phonetic_us": "美式IPA音标",
  "example_sentence_en": "英文例句",
  "example_sentence_zh": "中文翻译"
}

单词/词组：{selected_text}
```

#### 浮框显示

- 在选中文本附近显示浮框（使用绝对定位）
- 浮框包含：
- 单词/词组（英文，加粗，较大字号）
- 音标（IPA格式，英式/美式，灰色小字）
- 语音播放按钮（点击调用 `POST /api/wordbook/pronounce` 获取音频并播放）
- 中文解释（词性 + 释义，多行文本，自动换行）
- 例句（英文例句 + 中文翻译，灰色小字）
- 右上角五角星按钮（未收藏时空心，已收藏时实心金色）
- 点击浮框外部区域或按下 ESC 键关闭浮框
- 浮框支持拖拽移动位置（可选）

#### 收藏功能

- 点击五角星按钮，调用 `POST /api/wordbook/add` 接口
- 添加成功后，五角星变为实心金色
- 显示 toast 提示"已加入单词本"
- 如果单词已存在，显示提示"单词本中已存在该单词"

### 5. 单词发音实现逻辑

#### TTS API调用

- 使用DashScope TTS API生成单词发音
- API端点：`https://dashscope.aliyuncs.com/api/v1/services/audio/tts/single`
- 请求参数：
- `model`: "cosyvoice-v1"
- `input`: `{"text": "{word}"}`
- `parameters`: `{"voice": "longxiaochun", "format": "wav"}`
- 响应：音频数据（WAV格式）
- 前端接收后使用 `Audio` 对象播放

#### 备选方案

- 如果不想调用外部API，可以使用浏览器内置的 `SpeechSynthesis API`
- 实现简单，但发音质量可能不如专业TTS API

### 6. 性能优化考虑

#### 前端

- 文本选中事件使用防抖处理（300ms延迟），避免频繁触发API调用
- 单词解释结果缓存到内存（使用 `Map` 结构），相同单词不重复请求
- 单词本列表使用普通渲染（如果单词数量 < 100），超过100考虑虚拟滚动
- 音频播放使用单例模式，避免创建多个 `Audio` 对象

#### 后端

- 数据库操作使用索引优化查询性能（`user_id` 和 `word` 字段）
- 单词解释API添加简单缓存（内存缓存，key为单词文本），缓存时间1小时
- 数据库连接使用单例模式，避免重复创建连接
- 使用参数化查询防止SQL注入

### 7. 实现注意事项

#### 向后兼容性

- 现有API和功能保持不变
- 数据库表创建时检查是否已存在，避免重复创建
- 新增功能不影响现有对话功能
- 未登录用户也可以使用聊天功能（可选，根据需求决定）

#### 错误处理

- 网络请求失败时显示友好错误提示（使用现有的 `Toast` 组件）
- 数据库操作失败时回滚并返回错误信息
- 大模型API调用失败时返回默认错误信息
- 音频播放失败时显示提示"播放失败，请重试"

#### 安全性

- 模拟登录不做密码验证，但需要对用户名进行合法性校验（长度1-50字符，只允许字母、数字、下划线）
- API接口添加基本的参数校验（使用Pydantic）
- 数据库操作使用参数化查询，防止SQL注入
- 用户输入进行转义处理，防止XSS攻击

#### 用户体验

- 登录页面自动聚焦到输入框
- 支持回车键提交登录表单
- 单词解释浮框显示加载动画
- 音频播放按钮显示播放状态（播放中显示暂停图标）
- 删除单词时显示确认弹窗
