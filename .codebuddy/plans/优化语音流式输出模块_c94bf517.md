## 产品概述

修复 Omni 项目前端语音流式输出无声音的问题，并移除 TTS 相关代码，统一使用多模态大模型直接返回的音频输出。优化后，智能体回复时应立即开始按顺序播放语音。

## 核心功能

- 修复 `PcmPlayer` 中 `AudioContext` 未正确激活导致无声音的问题（`prepare()`/`ensureCtx()` 未 await `resume()`）
- 将 `PcmPlayer` 的关键方法改为异步，确保 AudioContext 在 `running` 状态后再调用 `source.start()`
- 移除所有 TTS 相关代码（前端 UI、API 调用、后端路由），统一使用多模态大模型直接返回的音频
- 保留语音回放功能（`voiceAudioUrls` 相关逻辑不变）
- 智能体回复时立即开始按顺序播放语音（流式播放体验优化）

## 技术栈

- 前端：React + TypeScript + Tailwind CSS（现有栈，不改动）
- 后端：Python + FastAPI（只移除 TTS 路由，不改音频生成架构）

## 实现方案

### 根本原因

`pcmPlayer.ts` 中 `prepare()` 和 `ensureCtx()` 调用了 `this.ctx.resume()` 但未 await，导致 `resume()` Promise 未完成时 `ctx.state` 仍为 `"suspended"`，后续 `source.start()` 调用失败（异常被 catch 静默处理），导致无声音。

### 修改策略

#### 1. `frontend/src/audio/pcmPlayer.ts` — 核心修复

- **`prepare()` 改为 `async`**：await `this.ctx.resume()`，确保 AudioContext 激活后再返回
- **`ensureCtx()` 改为 `async`**：await `this.ctx.resume()`，确保返回的 context 处于 `running` 状态
- **`enqueuePcm16Base64()` 改为 `async`**：await `ensureCtx()`，确保 context 激活后再创建 AudioBuffer
- **`playNext()` 添加防御性检查**：在 `source.start()` 前检查 `ctx.state`，若为 `"suspended"` 则先 `resume()` 并在 then 中调用 `start()`，作为双重保障

关键代码修改：

```typescript
// prepare() 改为 async
async prepare(sampleRate = 16000): Promise<void> {
    // ...
    if (this.ctx.state === "suspended") {
        await this.ctx.resume();
    }
    // ...
}

// ensureCtx() 改为 async
private async ensureCtx(): Promise<AudioContext> {
    // ...
    if (this.ctx.state === "suspended") {
        await this.ctx.resume();
    }
    return this.ctx;
}

// enqueuePcm16Base64() 改为 async
async enqueuePcm16Base64(b64: string, sampleRate = 16000): Promise<void> {
    // ...
    const ctx = await this.ensureCtx();
    // ...
}

// playNext() 中添加防御性检查
private playNext(ctx: AudioContext, triggerSeq: number) {
    // ...
    const doStart = () => {
        try { source.start(); } catch (e) { this.log(`start() 异常: ${e}`); }
    };
    if (ctx.state === "suspended") {
        ctx.resume().then(() => doStart());
    } else {
        doStart();
    }
}
```

## 架构设计

### 数据流（修复后）

```
用户语音输入 → MediaRecorder → 后端 /api/chat/voice
→ Omni 多模态模型（stream_call）→ SSE 流式返回
→ 前端 handleSse → playerRef.current.enqueuePcm16Base64()
→ PcmPlayer 按顺序播放（AudioContext 已正确激活）
```

### 关键保障

1. `prepare()` 在用户手势中调用并 await，确保 AudioContext 激活
2. `ensureCtx()` 在每次 enqueue 时检查并 await resume
3. `playNext()` 中双重检查 context 状态，防止 suspended 状态下调用 start()
