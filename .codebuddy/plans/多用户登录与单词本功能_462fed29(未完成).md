## 用户需求概述

增加一个多用户登录模拟系统、单词解释模块和用户单词本模块到现有的 Omni Web Chat 应用中。

## 产品概述

基于现有微信风格 Web 智能体对话应用，新增用户系统和学习辅助功能。用户可以通过简单用户名输入模拟登录，在对话过程中选中英文文本获取单词解释，并将生词加入个人单词本进行统一管理。

## 核心功能

### 1. 模拟多用户登录

- 应用启动时显示登录页面，用户输入用户名即可登录
- 登录状态通过 localStorage 持久化存储
- 支持切换用户（退出登录后重新输入用户名）
- 不做实际认证，用户名唯一标识用户

### 2. 单词解释模块

- 用户在聊天界面选中英文文本时，自动弹出浮框显示单词/词组的中文解释
- 解释内容通过调用现有大模型 API（DashScope Omni）生成
- 浮框右上角显示五角星按钮，点击可将单词加入个人单词本
- 浮框支持拖拽、关闭，不阻碍用户继续操作

### 3. 用户单词本模块

- 基本的增删查功能
- 单词列表按添加时间倒序排列
- 支持删除已收藏的单词
- 显示单词、释义、添加时间

### 4. UI 入口

- 聊天页面右上角添加单词本图标按钮
- 点击按钮进入单词本列表页面

## 数据存储

- 使用 SQLite 数据库持久化存储用户信息和单词本数据
- 后端新增数据库初始化和 CRUD 接口

## 技术栈选择

### 前端技术栈

- **框架**: React 19 + TypeScript（与现有项目一致）
- **样式**: Tailwind CSS v4（与现有项目一致）
- **路由**: React Router v7（与现有项目一致）
- **组件库**: 不引入额外组件库，使用自定义组件 + Tailwind CSS（保持项目轻量）

### 后端技术栈

- **框架**: FastAPI + Python 3.11+（与现有项目一致）
- **数据库**: SQLite（Python 内置 sqlite3 模块，无需额外依赖）
- **API 设计**: RESTful API，与现有 API 风格保持一致

## 实现方案

### 1. 数据库设计

#### users 表

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | INTEGER | 主键，自增 |
| username | TEXT | 用户名，唯一 |
| created_at | TEXT | 创建时间（ISO 格式） |


#### wordbook 表

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | INTEGER | 主键，自增 |
| user_id | INTEGER | 外键，关联 users.id |
| word | TEXT | 单词/词组 |
| explanation | TEXT | 中文解释 |
| example_sentence | TEXT | 例句（可选） |
| created_at | TEXT | 添加时间（ISO 格式） |
| UNIQUE(user_id, word) |  | 防止同一用户重复添加同一单词 |

#### API 端点设计

**用户相关**:

- `POST /api/users/login` - 模拟登录（创建或获取用户）
- `GET /api/users/me?username=xxx` - 获取当前用户信息

**单词本相关**:

- `POST /api/wordbook/add` - 添加单词到单词本
- `GET /api/wordbook/list?user_id=xxx` - 获取用户单词本列表
- `DELETE /api/wordbook/{word_id}` - 删除单词本中的单词
- `POST /api/wordbook/explain` - 获取单词/词组的中文解释（调用大模型 API）

### 4. 单词解释实现逻辑

#### 文本选中监听

- 在 `ChatPage` 组件中监听 `mouseup` 事件
- 检测是否有文本被选中
- 判断选中文本是否为英文（通过正则匹配）
- 获取选中文本的位置信息（用于定位浮框）

#### 解释生成

- 调用后端 `POST /api/wordbook/explain` 接口
- 后端使用现有 `omni_client` 调用大模型 API 生成解释
- Prompt 设计：`解释以下英文单词/词组的中文含义，包括词性、中文释义，如果是词组请给出例句：{selected_text}`

#### 浮框显示

- 在选中文本附近显示浮框
- 浮框包含：单词/词组、中文解释、五角星收藏按钮
- 点击浮框外部或按下 ESC 键关闭浮框
- 浮框支持拖拽移动位置

### 5. 性能优化考虑

#### 前端

- 文本选中事件使用防抖处理，避免频繁触发
- 单词解释结果缓存到内存（Map 结构），相同单词不重复请求
- 单词本列表使用虚拟滚动（如果单词数量较多）

#### 后端

- 数据库操作使用索引优化查询性能
- 单词解释 API 添加简单缓存（内存缓存，key 为单词文本）
- 数据库连接使用单例模式，避免重复创建连接

### 6. 实现注意事项

#### 向后兼容性

- 现有 API 和功能保持不变
- 数据库表创建时检查是否已存在，避免重复创建
- 新增功能不影响现有对话功能

#### 错误处理

- 网络请求失败时显示友好错误提示
- 数据库操作失败时回滚并返回错误信息
- 大模型 API 调用失败时返回默认错误信息

#### 安全性

- 模拟登录不做密码验证，但需要对用户名进行合法性校验（长度、特殊字符）
- API 接口添加基本的参数校验
- 数据库操作使用参数化查询，防止 SQL 注入

## 设计风格

采用与现有应用一致的微信风格设计，保持界面简洁、现代、易用。整体设计语言遵循 WeChat 的设计规范，使用圆角、阴影、半透明等视觉效果。

## 页面设计

### 1. 登录页面（LoginPage）

- **布局**: 居中卡片式布局，白色背景，圆角，阴影效果
- **顶部**: 应用图标和标题 "Omni Chat"
- **中间**: 用户名输入框（圆角边框，focus 时高亮）
- **底部**: "进入聊天" 按钮（绿色，WeChat 风格）
- **配色**: 背景浅灰色 #ededed，卡片白色 #ffffff，按钮绿色 #07c160

### 2. 单词本页面（WordbookPage）

- **顶部导航栏**: 左侧返回按钮，中间标题 "我的单词本"，右侧清空按钮（可选）
- **单词列表**: 
- 每个单词项卡片式布局，白色背景，底部分隔线
- 左侧显示单词（英文，加粗），下方显示中文解释
- 右侧显示添加时间（灰色小字）和删除按钮（红色垃圾桶图标）
- **空状态**: 当单词本为空时，显示提示文字 "暂无收藏的单词" 和引导图标
- **配色**: 与聊天页面保持一致

### 3. 单词解释浮框（WordExplanationPopup）

- **布局**: 小卡片式浮框，白色背景，圆角，阴影
- **内容区域**:
- 顶部：选中单词/词组（英文，加粗，较大字号）
- 中部：中文解释（多行文本，自动换行）
- 底部：例句（灰色小字，可选）
- **右上角**: 五角星按钮（未收藏时空心，已收藏时实心金色）
- **定位**: 跟随选中文本位置，自动调整避免超出屏幕
- **关闭**: 点击浮框外部区域或按下 ESC 键关闭

### 4. 聊天页面修改（ChatPage）

- **右上角**: 在现有设置按钮和人格下拉框之间，添加单词本书籍图标按钮
- **按钮样式**: 与其他图标按钮保持一致（圆形，hover 时背景变色）

## 交互设计

- **登录页面**: 输入框支持回车键提交，按钮点击有 ripple 效果
- **单词解释浮框**: 
- 显示/隐藏使用淡入淡出动画
- 五角星按钮点击有缩放动画
- 添加成功后显示 "已加入单词本" toast 提示
- **单词本页面**: 
- 删除按钮点击后有确认弹窗
- 列表项滑动删除（移动端）或点击删除（桌面端）
- **响应式设计**: 
- 桌面端：最大宽度 448px（与现有聊天界面一致）
- 移动端：全屏显示
